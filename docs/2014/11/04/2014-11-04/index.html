##Xcodebuild

当换一台机器的时候，你写的脚本还能正常跑起来吗？:-D 有心整理一下ios的打包脚本，更智能一点，省去哪些配置的过程。

有必要了解一下自动打包是如何实现的，那就是交给xcrun、xcodebuild。

  • xcodebuild负责将工程源文件编译成xxx.app
  • xcrun负责给xxx.app签名并打包成xxx.ipa

###签名的过程

  • 第一步清理工程： xcodebuild -targe targetName clean # 清理工程

  • 第二部编译工程： xcodebuild -target targetName build #编译工程 另[build]是默认参数

  • 第三部打包 打包的参数就是 xcrun -sdk iphoneos PackageApplication -v 源app路径 -o 输出的ipa路径 xcrun -sdk iphoneos PackageApplication -v $BuildPath/xxx.app -o xxx.ipa

  • 给应用签名（这两个命令都可以实现） xcodebuild -target targetName CODE_SIGN_IDENTITY="iPhone Distribution:XXXXXX" xcrun -sdk iphoneos PackageApplication -v 源app路径 -o 输出的ipa路径 --sign "iPhone Distribution:XXXXXX"

##新需求 工程有一些引用的路径不能够使用相对路径，当我的工程在另外一台机器上打包时就不能正常工作了，需要重新设置一下。%>_<%
